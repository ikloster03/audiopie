# Управление проектами

Проекты AudioPie позволяют сохранять работу и продолжать позже. Это особенно полезно для сложных аудиокниг, требующих нескольких сессий редактирования.

## Файлы проектов

Проекты AudioPie сохраняются как файлы `.audiopie`. Эти файлы на основе JSON содержат:

- Список и порядок треков
- Конфигурацию глав
- Метаданные (название, автор и т.д.)
- Ссылку на обложку
- Настройки, специфичные для проекта

## Создание проекта

### Новый проект

1. Запустите AudioPie
2. Нажмите **Новый проект** в Менеджере проектов
3. Начните добавлять треки и редактировать

### Из существующих файлов

1. Запустите AudioPie
2. Нажмите **Новый проект**
3. Перетащите ваши MP3-файлы
4. Сохраните проект

## Сохранение проектов

### Сохранить

Нажмите `Ctrl+S` или кнопку **Сохранить** (значок диска):

- Первое сохранение: Открывает диалог для выбора места
- Последующие сохранения: Сохраняет в то же место

### Требования для сохранения

Чтобы сохранить проект, вам нужно:
- ✅ Хотя бы один трек
- ✅ Название в метаданных

Если эти условия не выполнены, вы увидите уведомление об ошибке.

### Автосохранение

В настоящее время AudioPie не имеет автосохранения. Не забывайте регулярно сохранять работу!

## Открытие проектов

### Из Менеджера проектов

1. Запустите AudioPie
2. Нажмите **Открыть проект**
3. Выберите ваш файл `.audiopie`
4. Нажмите **Открыть**

### Из файлового менеджера

Вы также можете дважды кликнуть на файл `.audiopie` в файловом менеджере, чтобы открыть его в AudioPie (если настроены ассоциации файлов).

## Структура проекта

### Что сохраняется

| Данные | Сохраняется |
|--------|-------------|
| Пути к трекам | ✅ |
| Порядок треков | ✅ |
| Названия глав | ✅ |
| Порядок глав | ✅ |
| Все метаданные | ✅ |
| Путь к обложке | ✅ |

### Что НЕ сохраняется

| Данные | Сохраняется |
|--------|-------------|
| Аудиоданные | ❌ (ссылки, не встроено) |
| Результат сборки | ❌ |
| Глобальные настройки | ❌ (хранятся отдельно) |

:::important
Файлы проектов ссылаются на ваши аудиофайлы по пути. Если вы переместите или удалите оригинальные MP3-файлы, проект не сможет их найти.
:::

## Закрытие проектов

Чтобы закрыть текущий проект:

1. Нажмите кнопку **Закрыть** (значок X) в заголовке
2. Вы вернётесь в Менеджер проектов

:::caution
Несохранённые изменения будут потеряны! AudioPie предупредит вас, если есть несохранённая работа.
:::

## Переносимость проектов

### Перенос проектов

Если вам нужно перенести проект на другой компьютер:

1. Скопируйте файл `.audiopie`
2. Скопируйте все связанные MP3-файлы
3. Сохраните ту же структуру папок, ИЛИ
4. Обновите пути после открытия на новом компьютере

### Лучшая практика

Храните файлы проектов вместе с аудио:

```
Моя аудиокнига/
├── Моя аудиокнига.audiopie
├── треки/
│   ├── 01 - Введение.mp3
│   ├── 02 - Глава первая.mp3
│   └── ...
└── обложка.jpg
```

## Советы по работе с проектами

### Соглашения об именовании

Используйте описательные имена проектов:

```
Хорошо:
- "Великий Гэтсби - Фицджеральд.audiopie"
- "Дюна Книга 1.audiopie"

Избегайте:
- "project1.audiopie"
- "audiobook.audiopie"
- "untitled.audiopie"
```

### Контроль версий

Для сложных проектов рассмотрите хранение версий:

```
Проект Аве Мария v1.audiopie    (начальный импорт)
Проект Аве Мария v2.audiopie    (главы отредактированы)
Проект Аве Мария final.audiopie (готов к сборке)
```

### Стратегия резервного копирования

Делайте резервные копии:
- Ваших файлов проектов `.audiopie`
- Ваших исходных MP3-файлов
- Ваших изображений обложек

## Устранение неполадок

### Ошибки «Файл не найден»

Если треки показывают ошибки после открытия:
- Исходные MP3-файлы были перемещены или удалены
- Восстановите файлы в исходное место, ИЛИ
- Удалите и заново добавьте треки

### Проект не открывается

- Файл может быть повреждён
- Попробуйте открыть в текстовом редакторе для проверки валидности JSON
- Проверьте, что это валидный файл `.audiopie`

### Изменения не сохраняются

Убедитесь, что:
- У вас есть права на запись в место сохранения
- Диск не заполнен
- Файл не открыт в другом приложении
